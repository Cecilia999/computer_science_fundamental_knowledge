# 抽象类与抽象方法 - 关键字 abstract

随着继承层次中ー个个新子类的定义,类变得越来越具体,而父类则更 般,更通用。类的设计应该保证父类和子类能够共享特征。有时将一个父 类设计得非常抽象,以至于它没有具体的实例,这样的类叫做**抽象类**。

## 1. 抽象类与抽象方法的使用、注意事项

```java
package com.ming.objectOrientedPprogramming;

/**
 * @Author: MingChen
 * @Date: 2021/8/20
 *
 * abstract关键字的使用
 * 1. abstract:抽象的
 * 2. abstract可以用来修饰的结构:类、方法
 *
 * 3. abstract修饰类:抽象类
 *      > 此类不能实例化
 *      > 抽象类中一定有构造器,便于子类实例化时调用(涉及:子类对象实例化的全过程)
 *      > 开发中,都会提供抽象类的子类,让子类对象实例化,完成相关的操作
 *
 * 4. abstract修饰方法:抽象方法
 *      > 抽象方法只有方法的声明,没有方法体
 *      > 包含抽象方法的类,一定是一个抽象类。反之,抽象类中可以没有抽象方法
 *      > 若子类重写了父类中的所有的抽象方法后,此子类方可实例化
 *        若子类没有重写父类中的所有的抽象方法,则子类也是一个抽象类,需要使用 abstract修饰
 *
 * 5. abstract使用上的注意点:
 *      > abstract不能用来修饰:属性、构造器等结构
 *      > abstract不能用来修饰: private私方法、static静态方法、final的方法、final的类
 */

public class AbstractTest {
    public static void main(String[] args) {
        //一旦Person类被abstract修饰了，就不能实例化
//        Person p = new Person();
//        p.eat();

        Student student = new Student("小明", 16);
        student.eat();
    }

}

abstract class Person{
    String name;
    int age;

    public Person(){

    }

    public Person(String name, int age){
        this.name = name;
        this.age = age;
    }

    //不是抽象方法
//    public void eat(){
//        System.out.println("人吃饭");
//    }
    //抽象方法
    public abstract void eat();

    public void walk(){
        System.out.println("人走路");
    }
}

class Student extends Person{
    //调用父类Person的构造器
    public Student(String name, int age){
        super(name, age);
    }

    public void eat(){
        System.out.println("学生吃饭");
    }
}
```
